Debugging Javascript

  • The browser will try and make up for mistakes in HTML and CSS.
  • Mistakes in JavaScript will cause it to fail utterly.
  • So don't make any mistakes.
  • Or, learn how to debug your code.

Debugging Javascript

  • HTML and CSS should have taught you that syntax is extremely important.
  • The same goes for JS, but the console may help you out.
  • Your browser has a console (ALT+CMD+J in Chrome).
  • Errors will show in the console, so if something isn't working, check there.
  • You can also write code directly into the console! Try typing in 1+2 and see what happens.

State.

"... a state is a unique configuration of information in a program or machine."

- http://en.wikipedia.org/wiki/State_(computer_science)

Altering states.

  • When a web page first loads, it's in it's initial state.
  • Without javascript, links jump from page to page, seeing separate initial states.
  • With javascript, links (and a lot of other things) can alter the state of the current page we're viewing.
  • ex. lightboxes, carousels, these lecture slides, etc.
  • State is almost always changed via a user interaction.

Interaction - How?

Your JS hijacks your HTML.

How we interact w/ JS.

  1. We choose an element in our HTML that we want to be a point of interaction.
  2. We attach an event listener to that element with jQuery, choosing the event (click, hover) on the HTML element we wish will intitiate the interaction.
  3. We then write a function which will execute after the interaction's chosen event has occured in the browser.
  4. ex:
    $(function() {
    $('#some-link-element').click(function(e) {
        alter('explosions.');
    }); 
});

A few reminders.

  • Uses a script tag to load external Javascript files. Always load jQuery first.
  • <script src="jquery.js"></script>
  • For this class, your javascript should be encapsulated as follows:
  • $(function() {
   // your code goes here. 
});
  • Hiding an element with jQuery:
  • $(function() {
    $('#id-of-element').hide();
});

Javascript and Forms.

But first, forms.

HTML Forms

The form element represents a collection of form-associated elements, some of which can represent editable values that can be submitted to a server for processing. 
<form action="some/url/" method="get">

Form attributes.

  • action="some/url/"
    The URL to which the form's data is sent. The user will also end up at this address.
  • method="get"
    For our purposes, just use get.

    What it refers to is the request method (get or post).

The <input> tag.

<input type="text" id="query" name="query">
  • Creates a single-line field for text input.

The <textarea> tag.

<textarea name="message"></textarea>
  • Creates a multi-line field for text input.
  • Note: Textareas have closing tags, unlike inputs!

Naming fields with the <label> tag.

<label for="query">Search Term</label>
<input type="text" id="query" name="query">
  • The for attribute matches the id attribute of an input.

The <input> tag, cont'd.

<label>
<input type="checkbox" name="foo">
Are you sure?
</label>
  • Creates a checkbox.
  • Labels can be wrapped around inputs, which can be useful for checkboxes and radio buttons.

The <input> tag, cont'd.

<label><input type="radio" name="bar" value="a"> Option A</label>
<label><input type="radio" name="bar" value="b"> Option B</label>
  • Creates two radio buttons.
  • Sets are defined using the name attribute.
  • Only one member of a set can be selected at once.

The <select> tag.

<label for="choose">Make a choice.</label>
<select id="choose" name="choose">
<option>Choice A</option>
<option>Choice B</option>
<option>Choice C</option>
</select>
  • Similar to radio buttons by default.
  • Other attributes exist that will allow multi-selection and option grouping.

The <button> tag.

<button>Send!</button>
  • By default, buttons will submit the form to its action using its method.
  • Also possible: <input type="submit"> or <input type="button">.

Forms and CSS

  • As a general guideline, avoid over-styling form elements as they can have unpredictable consequences across devices.
  • Also, the more customized form elements become, the less obvious things are for your user as conventions are broken.
  • All form elements have display: inline; by default, but you can use float or different HTML structures to change that.

Forms and JavaScript.

Forms and Javascript

  • Target the form element using jQuery then attach a submit event.
  • Call e.preventDefault to stop the form from submitting via the browser.
$(function() {

    $('form').submit(function(e) {

        // prevent the form from doing default stuff...
        e.preventDefault();

        // do whatever you want...

    });

})
