Debugging Javascript

  • The browser will try and make up for mistakes in HTML and CSS.
  • Mistakes in JavaScript will cause it to fail utterly.
  • So don't make any mistakes.
  • Or, learn how to debug your code.

Debugging Javascript

  • HTML and CSS should have taught you that syntax is extremely important.
  • The same goes for JS, but the console may help you out.
  • Your browser has a console (ALT+CMD+J in Chrome).
  • Errors will show in the console, so if something isn't working, check there.
  • You can also write code directly into the console! Try typing in 1+2 and see what happens.

State.

"... a state is a unique configuration of information in a program or machine."

- http://en.wikipedia.org/wiki/State_(computer_science)

Altering states.

  • When a web page first loads, it's in it's initial state.
  • Without javascript, links jump from page to page, seeing separate initial states.
  • With javascript, links (and a lot of other things) can alter the state of the current page we're viewing.
  • ex. lightboxes, carousels, these lecture slides, etc.
  • State is almost always changed via a user interaction.

Interaction - How?

Your JS hijacks your HTML.

How we interact w/ JS.

  1. We choose an element in our HTML that we want to be a point of interaction.
  2. We attach an event listener to that element with jQuery, choosing the event (click, hover) on the HTML element we wish will intitiate the interaction.
  3. We then write a function which will execute after the interaction's chosen event has occured in the browser.
  4. ex:
    $(function() {
    $('#some-link-element').click(function(e) {
        alter('explosions.');
    }); 
});

A few reminders.

  • Uses a script tag to load external Javascript files. Always load jQuery first.
  • <script src="jquery.js"></script>
  • For this class, your javascript should be encapsulated as follows:
  • $(function() {
   // your code goes here. 
});
  • Hiding an element with jQuery:
  • $(function() {
    $('#id-of-element').hide();
});
