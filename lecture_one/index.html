What really happens when you go to www.google.com/mail. (review)

  1. A request is made for "www.google.com" to your local name server. Notice the lack of "/mail".
  2. The physical IP address (where the google server is actually located) is returned to your web browser.
  3. Your web browser then sends an http request to that server, including that it wants "/mail" specifically.
  4. The server recognizes that you want "/mail," and returns the HTML needed for your browser to render GMail.
