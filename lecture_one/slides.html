What really happens when you visit this site.

What really happens when you visit this site.

  1. You type in the above URL (http://sankhomallik.com/icl-spring-2012/lecture_one/slides.html) and hit enter.
  2. The browser makes a request to the server, or sankhomallik.com.
  3. The request specifically asks for a file, or /icl-spring-2012/lecture_one/slides.html.
  4. The server looks internally for a file named slides.html within the folders /icl-spring-2012/lecture_one/ and returns that file to your browser.

Cool story bro. But where do I come in?


Your browser requests files from a server.

The server returns HTML.

You write the HTML files.

  • For me, it would be: http://webspace.newschool.edu/~malliks/

Accessing your web space.

  • Your web space is accessed via SFTP, which is another way for computers to talk over the Internet.
  • We will be using an application called Fetch.

Using Fetch to make an SFTP connection.

After opening Fetch, you should be presented with a "New Connection" window. Here's how to use it:

  1. Hostname: webspace.newschool.edu
  2. Username: Your NetID (e.g. oneillp)
  3. Connect Using: SFTP
  4. Password: Your NetID password.
